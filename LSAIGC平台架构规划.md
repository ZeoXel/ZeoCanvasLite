# LSAIGC 平台架构规划

## 一、背景与现状

公司当前运营基于 1.0 架构的 AI 智能平台，由门户网站通过 iframe 聚合多个独立子系统。经过一个月的开发，2.0 版本已完成用户体系、支付系统、存储系统的数据互通迁移，具备独立服役能力。

现需对两个版本的职能进行明确划分，使其各司其职、协同运作，服务于公司三大核心业务场景。

---

## 二、平台核心职责

本平台体系承担三项核心职责：

**1. 服务 B/C 端用户**
B 端用户通过预设的成品工作流直接完成业务需求（如批量生成营销素材、自动化视频产出等）；C 端用户直接触达商用大模型，按需生成图片、视频、音乐等内容产出。

**2. AI 影视创作课程的实践平台**
作为公司 AI 影视创作培训课程的配套实践工具，学员在课程中学习理论知识后，直接在平台上进行实操练习，实现教学与实践一体化。

**3. 影视创作团队的专业工作站**
承载公司内部影视创作团队的日常生产任务，处理从概念设计到素材生成的完整创作流程，满足商业级产出要求。

---

## 三、双版本职能划分

### 3.1 整体架构

```
┌─────────────────────────────────────────────────────────────┐
│                     LSAIGC 平台体系                          │
├────────────────────────────┬────────────────────────────────┤
│      门户站（1.0 演进）      │       创作工作站（2.0）          │
│                            │                                │
│   定位：展示 + 服务 + 教学   │   定位：专业生产工具             │
│   受众：B/C 端用户、学员     │   受众：学员实操、创作团队        │
│   形态：多页面聚合式         │   形态：一体化画布式             │
└────────────────────────────┴────────────────────────────────┘
```

### 3.2 门户站（1.0 演进）

**定位：公司对外窗口 + 业务服务入口 + 课程教学中心**

门户站承担所有面向外部的展示与服务职能，是用户接触平台的第一入口。

| 板块 | 功能说明 |
|------|---------|
| 首页 | 公司介绍、品牌文化展示、产品能力概览、成功案例展示 |
| 课程教学 | AI 影视创作课程体系，包含课程列表、章节内容、视频教程、学习进度管理 |
| 工作流广场 | 基于 Coze 构建的成品工作流展示，B 端用户根据预设场景直接使用，C 端用户按需调用 |
| 用户中心 | 注册登录（手机 + 短信验证）、个人信息管理、余额与积分管理 |
| 充值系统 | 支付宝 / 微信支付充值，订单管理，交易记录 |
| 推广系统 | 推广人注册、推广码生成、奖励统计 |

**关键特征：**

- 多页面结构，信息层次分明，适合浏览和了解
- 保留 iframe 方式集成外部服务（工作流广场对接 Coze）
- 承载完整的用户体系和支付基础设施
- 课程板块与 2.0 工作站形成"理论—实践"闭环

### 3.3 创作工作站（2.0）

**定位：一体化 AI 媒体创作引擎 + 课程实操平台 + 团队生产工作站**

工作站是平台的核心生产力工具，舍弃传统的分散式页面布局，以统一画布为基础，将所有 AI 创作能力以节点形式组织，打通各功能间的素材可视化连接。

| 能力层 | 节点类型 | 说明 |
|--------|---------|------|
| 图像生成 | Midjourney、DALL-E、Flux、Ideogram | 文生图、图生图、垫图、局部重绘、高清放大 |
| 视频生成 | Luma、Runway、Kling、Pika、Viggle、MiniMax | 文生视频、图生视频、动画生成 |
| 音频/音乐 | Suno、Udio、TTS、Whisper | 音乐生成、语音合成、语音识别 |
| 对话/文本 | GPT、Claude、Deepseek、Gemini | 文案生成、脚本撰写、内容优化 |
| 3D 建模 | Tripo3D | 文本/图像生成 3D 模型 |
| 辅助工具 | 图像编辑、格式转换、素材库 | 创作流程中的辅助处理 |

**核心优势——画布式节点编排：**

```
┌──────────────────────────────────────────────────────────────┐
│                        统一画布                               │
│                                                              │
│   [文案节点] ──→ [分镜节点] ──→ [图像生成节点]                  │
│       │                              │                       │
│       │                              ▼                       │
│       │                        [视频生成节点] ──→ [输出]       │
│       │                              ▲                       │
│       └──────→ [音乐生成节点] ────────┘                       │
│                                                              │
│   ● 每个节点 = 一个 AI 能力                                    │
│   ● 节点间连线 = 素材自动流转                                   │
│   ● 整个画布 = 一条完整的创作工作流                              │
└──────────────────────────────────────────────────────────────┘
```

**与旧版创作空间的本质区别：**

旧版创作空间中，每个 AI 工具是独立页面，用户在不同工具间切换时需要手动保存、下载、再上传素材。工具之间没有数据连接，无法看到完整的创作链路。

新版画布将所有工具统一到一个可视化空间中，节点的输出自动成为下一个节点的输入，素材流转全程可视、可追溯、可复用。这不是界面改版，而是创作范式的升级。

## 四、双版本协同关系

### 4.1 数据层互通

两个版本共享同一套 Supabase 数据库，核心数据表完全互通：

| 共享数据 | 说明 |
|---------|------|
| 用户表（users） | 统一账号体系，一次注册两端通用 |
| 支付/余额表（payments / balance_logs） | 门户充值，工作站消耗，余额实时同步 |
| API Key 池（api_keys） | 统一的密钥分配与管理 |
| 使用统计（api_usage_stats） | 跨平台的用量汇总 |

用户在门户站注册充值后，即可在工作站中直接使用，无需重复登录或二次配置。

### 4.2 业务流闭环

```
门户站                              创作工作站
  │                                    │
  ├─ 用户注册/充值 ──────────────────→ 余额可用
  │                                    │
  ├─ 课程学习（理论） ───"前往实操"──→ 画布实践（实操）
  │                                    │
  ├─ 工作流广场（B端成品流程）          │
  │                                    ├─ 画布创作（专业生产）
  ├─ 公司展示/案例 ←── 产出成果回流 ───┤
  │                                    │
  └─ 推广/获客 ────→ 新用户 ──→ 进入体系
```

### 4.3 用户旅程示例

**场景 A：C 端个人用户**
用户通过门户站了解平台 → 注册并充值 → 在工作流广场选择"AI 短视频生成"模板一键使用 → 如需深度定制，进入工作站画布自行编排节点。

**场景 B：B 端企业客户**
企业通过门户站了解公司业务和案例 → 联系合作 → 使用工作流广场中的定制化工作流完成批量任务 → 高阶需求由创作团队在工作站中处理。

**场景 C：课程学员**
学员在门户站报名课程 → 观看理论教学视频 → 点击"前往实操"进入工作站 → 在预设的课程画布模板中跟随教程完成练习 → 画布本身即为课程内容的可视化呈现。

**场景 D：内部影视创作团队**
团队成员直接进入工作站 → 创建/打开项目画布 → 从文案分镜到图像视频音乐全链路在画布中完成 → 素材自动流转，流程可复用可迭代 → 最终输出生产级成品。

## 五、技术架构

### 5.1 技术栈对照

| 维度 | 门户站（1.0 演进） | 创作工作站（2.0） |
|------|-------------------|------------------|
| 框架 | Next.js 15 + React 19 | Next.js + React 19 |
| 样式 | Tailwind CSS 4 | Tailwind CSS |
| 认证 | NextAuth.js（手机 + SMS） | NextAuth.js（共享会话） |
| 数据库 | Supabase（PostgreSQL） | Supabase（同一实例） |
| 支付 | 支付宝 SDK + 微信 APIV3 | 复用门户支付系统 |
| 短信 | 腾讯云 SMS | 复用门户验证 |
| 部署 | Docker / 独立域名 | Docker / 独立域名 |

```
                    ┌─────────────┐
                    │   Nginx     │

              ┌────────────┼────────────┐

     ┌────────▼───┐  ┌─────▼─────┐  ┌──▼──────────┐
     │  门户站     │  │ 创作工作站 │  │  Coze 工作流 │
     │ portal.    │  │ studio.   │  │  flow.      │
     │ lsaigc.com │  │ lsaigc.com│  │  lsaigc.com │
     └────────┬───┘  └─────┬─────┘  └─────────────┘
              │            │
              └──────┬─────┘
                     │
              ┌──────▼──────┐
              │  Supabase   │
              │ (PostgreSQL) │
              │  共享数据层   │
              └──────┬──────┘
                     │
              ┌──────▼──────┐
              │  API 网关    │
              │ api.lsaigc  │

              └─────────────┘
```

### 5.3 核心数据模型

```
users
  ├── id, phone, name, role, status
  ├── balance, total_recharge_amount
  └── created_at, updated_at

api_keys
  ├── id, key_value, provider, model_type
  ├── assigned_user_id, quota_limit
  └── status (active/assigned/suspended)

payments
  ├── order_no, user_id, amount, points
  ├── payment_method (alipay/wechat)
  └── status (pending → paid → refunded)

balance_logs
  ├── user_id, type (recharge/consume/bonus/...)
  ├── amount, balance_before, balance_after
  └── description, created_at
```

## 六、演进路线

### 第一阶段：双版本上线（1-2 周）

**目标：两个版本各自独立服役，职能清晰划分**

- 门户站：保留现有功能，强化首页展示与工作流广场
- 创作工作站：补齐剩余待接入的模型节点，完善画布交互体验
- 数据层：确认双版本共享 Supabase 实例的稳定性
- 导航互通：门户站增加"进入工作站"入口，工作站保留"返回门户"链接

### 第二阶段：课程体系接入（1-2 个月）

**目标：建立"理论 → 实操"的教学闭环**

- 门户站：上线课程教学板块（课程列表、章节视频、学习进度）
- 创作工作站：为每个课程章节预设对应的画布模板，学员一键加载即可跟随教程实操
- 打通课程进度与工作站实操记录的数据关联

### 第三阶段：生产力深化（2-4 个月）

**目标：将工作站打磨为真正的专业生产工具**

- 画布模板库：沉淀团队高频使用的创作流程为可复用模板
- 项目管理：支持多项目/多画布管理，团队协作
- 批量处理：支持节点批量执行，适应规模化生产需求
- 素材管理：统一的素材库，跨项目复用

### 第四阶段：生态扩展（4-6 个月）

**目标：从内部工具向开放平台演进**

- 模板市场：优质画布模板对外开放，B 端用户可直接购买使用
- API 开放：核心能力以 API 形式对外提供
- 插件系统：支持第三方节点接入，扩展画布能力边界

---

## 七、总结

本架构的核心思路是**职能分离、数据互通、各取所长**：

门户站聚焦"展示与连接"——让外部用户了解公司、学习课程、使用成品工作流，是公司的门面和服务窗口。

创作工作站聚焦"生产与创造"——以画布为核心，将分散的 AI 能力统一为可视化的创作流程，是课程的实践场所、团队的生产工具。

两者通过共享数据层实现无缝协同，用户在门户充值、在工作站消耗，在门户学理论、在工作站做实操。既保留了 1.0 积累的用户体系和业务能力，又充分发挥了 2.0 画布架构的生产力优势。
